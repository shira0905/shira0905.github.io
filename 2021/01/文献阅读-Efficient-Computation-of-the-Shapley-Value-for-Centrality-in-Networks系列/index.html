笔记0219

20210219

- 时隔近一个月重新细读, 细看了一下在g1 case 公式和推导终于明白了为什么是exact closed form of Shapley value. 确实很妙.

- 这篇文章的写作风格也是有借鉴意义的.

- 可能可以思考一下这个想法可以怎么用到我自己的研究上去:

(1) 有很多关于影响力测量的measure

(2) centrality的度量在传统上是对单个节点的, 此处generalized到节点的集合, 并且可以算集合总的单个节点的所贡献的团队整体centrality (这个整体centrality也就是也正方程的值)

(3) 对于这类定义在subset nodes上的量化影响力概念的特征方程,每个节点的平均边际贡献, 也就是每个节点的Shapley value. 用这个度量来作为重要度/centrality是符合常理的. 所有节点作为grand coalition V对应的特征方程的值是|V|, 然后每个节点在其中的贡献大小作为其重要度/centrality的值,

(4) 计算方面,如果抛开Shapley的概念, 那么就是求看每个节点的编辑贡献值的期望over all the permutation. 最直接的方法是求出所有的permutation进行平均, 但是其实可以用另外的思维也就是topograph的结构推导出的exact closed form, 来进行计算. 并且因为这个clodes form只和它的邻居们有关, 也就意味着复杂度不高

Game 1

定义

$\nu_1(C)=$ # agents at most 1degree away
the fringe of a coalition $C \subseteq V(G)$ includes all nodes reachable from the coalition in at most one hop.

$ \{v \in V(G): v \in C \text{ or } \exists u \in C \text{ such that } (u,v)\in E(G) \}$ 或者我自己写的形式如下 $ \{v \in V(G): v \in C \text{ or } \exists u \in C \text{ such that } d(u,v)=1 \}$

推导

过程类似Game2的

Game 2

定义

$\nu(C)=$ # agents with at least k neighbors in C

Intuitively, in this model each node can become active if a monotone activation function reaches some threshold. 如果说C是activate的人, 那么对任意一个人来说, 如果他的邻居中有>=k个人是activate的, 那么这个人也将会被activate.

Note that this game reduces to game g1 for k =1. 只需要有一个共同邻居就能被count in.

分析

What is the necessary and sufficient condition for node $v_i$ to marginally contribute node $v_j \in N_G(v_i) \cup \{v_i\}$ to the value of the coalition $C$ ?

  • If $v_j = v_i$

    $N_G(v_j) \cap C < k$

  • If $v_j \neq v_i$

    • $v_j \notin C$ , and
    • $N_G(v_j)\cap C==k-1$

推导(见草稿纸)

If $v_j \neq v_i$

  • $v_j$ 在 $v_i$ 之后, 不然就已经是圈里人了 [对应 $v_j \notin C$]
  • v_i 之前 恰好有 k-1 个 v_j 的邻居 [对应 $N_G(v_j)\cap C==k-1$ ]
    • 少了则 v_i 加入C之后 v_j 还是不能入圈
    • 多了则 v_i已经是圈里人

所以这样的permutation分析如下:

  • 为 $N_G(v_j)\cup\{v_j\}$ 选坑位 【注意这里是包括了 $v_i$ 的】
  • 从 $v_j$ 的 除了 $v_i$ 之外的 邻居里面取出 $k-1$ 个放在 $v_i$ 之前 $ N_G(v_j)-1 \choose k-1 $
  • 排列 $(k-1)!(N_G(v_j)+1-k)!$
  • 其他坑位由非$N_G(v_j)\cup\{v_j\}$的无关节点随意排序 $(|V(G)|-|N_G(v_j)|-1)!$

If $v_j = v_i$

v_i 之前 恰好x 个 v_j 的邻居, x =0,1,…., k-1

  • 为 $N_G(v_j)\cup\{v_j\}$ 选坑位
  • 从 $v_j$ 的 邻居里面取出 $x$ 个放在 $v_j$ 之前 $ N_G(v_j) \choose x $
  • 排列 $x!(N_G(v_j)-x)!$
  • 其他坑位由非$N_G(v_j)\cup\{v_j\}$的无关节点随意排序 $(|V(G)|-|N_G(v_j)|-1)!$

综上, 可以得到expected (over permutation of $V(G)$) increase of $\nu$ after $v_i$ joining $C$

Game 3

定义

$\nu_3(C)=$ # agents at most $d_{cutoff}$ away

weighted graphs.

推导(见草稿纸)

类似 game 1

$SV_3(v_i) = \sum_{v_j \in N(v_i,d)\cup\{v_i\}} \frac{1}{1+deg(v_j,d)}$

算法复杂度分析

  1. For each node v in the network G, the extended neighborhood and its size are first computed using Dijkstra’s algorithm, in $O(|E|+|V|log|V|)$ time (Cormen, 2001). 【怎么是这个时间复杂度？】

  2. The results are then used to directly implement the above equation, which takes maximum time $O(|V|^2)$.

    In practice this step runs much faster because the worst case situation only occurs when every node is reachable from every other node within cutoff.

Overall the complexity is $O(|V||E| + |V|^2log|V|)$ ， 这里是因为第一步比第二步复杂， 用第一步的复杂度即可。

参考: https://www.cs.auckland.ac.nz/compsci220s1c/lectures/2016S1C/CS220-Lectures27-29.pdf

假设有n 个点m条边

  • 单源最短路径 $\forall w \geq 0$ , 用 Dijkstra 算法.

    greedy的思想, 初始化S加入start, 然后每一次都选距离S中的点们距离最短的点v, 将其加入S中 (S中的任意点x已经d[x])最优在之后不会再update了), 然后对于每个v的出度邻居u, 检查经过v的距离是不是会比原来的距离更段, 如果是的话则更新d[u].

  • 单源最短路径 $\exists w<0$ , 用 Bellman Ford算法

    进行 n-1 轮的逐条边更新,

    在每一轮, 对于每一条边(u,v), 检测 — 如果 $d(u)+d(u,v) \leq d(v)$ 那么更新 $d(v) = d(u)+d(u,v)$

    注意: 如果不存在负和圈的时候, 在m-1极其之前, 一定会收敛

    $O((n-1)*m)$

    最坏情况 边可能有 $n(n-1)/2$ 这个时候 $O(n^3)$

  • all pairs 最短路径 Floyd. 矩阵DP, t时刻, 考察点 $v_t$, 根据 $A^{v_{t-1}}$ 生成 $A^{v_{t}}$ , 也就是 $v_t$ 变为一个可经过点时候, 各个节点对的距离会不会有所生成改善. 所以时间复杂度为 $O(n^3)$

  • Prims 和 Kruskal 是最小生成树的算法, 前者是已选边的相邻边的greedy检测是否造成环, 后者是从边集里面greedy检测是否成环.
