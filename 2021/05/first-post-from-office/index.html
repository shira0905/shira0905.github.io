多机维护 Hexo

Posted on 2021-05-20 Edited on 2021-06-28 In Tech , web

多台电脑协作用 hexo 维护 github page。

step 1

在my mac上面的shiyuan本来就是一个repo， 名字是 shiyuan_site之前一直不知道.

Ignore 了

  • hexo的 node_modules 和 package-lock.json
  • public/

不知道是什么部署原理，反正重用这个repo就是了

step 2

在office Mac

显示安装了一系列环境 npm， hexo， 等等

知道hexo命令在各个目录下都没有报错

step 3

但是在git clone 下来的shiyuan 下运行hexo server会报错

所以我就直接按照提示

然后就可以直接 hexo s 并且运行自己的网站了

step 4

至于生成和部署， 没有额外操作， 直接跟在my mac一样， 就可以在github上面出现新文章了，

猜测原理：

  • 在配置文件里面已经有了
  • 所以generate就是生成public 在本地， 并不会影响到github.io的
  • deploy的是什么， 既然不是public的话， 那估计就是使用traviouIC，把source code generate成 .deploy*（这个文件夹和public内容应该是一样的）， 然后再把这个文件夹里面的内容在push 到.io哪个repo 覆盖
  • 所以说即使不适用 -g命令， 也可以部署上去，
