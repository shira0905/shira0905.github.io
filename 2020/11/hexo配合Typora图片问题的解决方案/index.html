Hexo配合Typora图片问题的解决方案

Posted on 2020-11-26 Edited on 2020-11-27 In Tech

一般我都是使用Typora编辑.md 文件，经常会 command+ctrl+shift+4 快捷键截图然后复制到文档里面，这时候Typora把图片自动把图片保存子啊某个奇奇怪怪的地方。问题在于，在本地预览的时候无法找到图片文件（虽然是本机的绝对路径），更不用说public到github page了。安排解决！

主要参考 Hexo博客写作与图片处理的经验 ， 感谢博主的经验分享！

Why?

目标

  • 使用Typora编写的时候能够实时看到图片
  • 本地使用hexo server浏览效果时，也能够看到图片
  • 图片和Markdown文件放一起都上传到GitHub pages。

How?

Hexo配置asset文件夹

首先修改 hexo 全局配置文件 _config.yml 中的配置：

这样在我们每次新建Markdown文件的时候，都会创建一个与文件同名的文件夹用于存放图片。

使用hexo generate生成静态文件后，可以观察到图片已经拷贝到了对应的目录下，也就是 public 目录下。

Typora插图时复制到指定路径

Typora支持将插入的图片文件拷贝到指定路径，通过Typora->偏好设置->图像，然后参照下图选择复制到指定路径将图片拷贝到与Markdown文件同名目录下，也就是 post_asset_folder: true 这一步所指定的文件夹。

现在在本地预览网站时已经可以看到啦！

部署到github page 试试吧

在本地运行时，因为不是靠静态/public，而是按照 /_post 来动态生成的：

但是现在浏览的时候还不能够看到图片，因为图片的路径是文件同名文件夹的相对路径（适用于本地渲染）：

而生成的静态文件夹下没有那个同名文件夹所以造成了访问404。

那篇文章的博主试了hexo-asset-image和hexo-simple-image 这两个插件发现还是有问题，就研究了一下源码对插件内容做了修改（感谢+膜拜），现在可以用他的这个新的插件来解决图片显示的问题。

然后就正常地 hexo s 或者 hexo g -d 就可以本地预览或者部署到github page上了！

Conclusion

  • 修改Hexo的配置文件 post_asset_folder: true
  • 修改Typora的图片复制到指定地址 ./${filename} , 也就是hexo new filename 的时候自动创建的asset文件夹。
  • 为了使图片地址对静态文件也适用，安装插件 npm install hexo-image-link --save
