Hexo next 解决版面效率问题

Posted on 2020-12-29 Edited on 2021-01-07 In Tech , web

解决版面效率问题：动画速度慢、分页、间距不美观

取消页面加载动画

https://www.jianshu.com/p/ed726bcf6767

在根目录下的/theme/next/_config.yml文件中搜索motion。将motion下的enable参数设置为false即可关闭页面动画

首页文章间距

https://blog.csdn.net/weixin_42024255/article/details/82814433

打开\themes\next\source\css\_schemes\Mist\_posts-expanded.styl ，各种 margin 都可以试着调一下。

每页文章篇数

http://theme-next.iissnan.com/faqs.html#%E8%AE%BE%E5%AE%9A%E5%BD%92%E6%A1%A3%E9%A1%B5%E9%9D%A2%E6%96%87%E7%AB%A0%E7%9A%84%E7%AF%87%E6%95%B0

在 Hexo 里可以为首页和归档页面设置不同的文章篇数，但可能需要安装 Hexo 插件。详细步骤如下。

  1. 使用 npm install --save 命令来安装需要的 Hexo 插件。

  2. 等待扩展全部安装完成后，在 站点配置文章 中，设定如下选项：

    index页面默认10篇每页。我就全部设成0就是不允许分页了，点上一页下一页也怪麻烦的。

字体间距等

http://renke666.top/2020/01/25/hexo%E6%9B%B4%E6%94%B9%E5%AD%97%E4%BD%93%E4%B8%8E%E6%AE%B5%E8%90%BD%E5%A4%A7%E5%B0%8F/

https://www.cnblogs.com/LyShark/p/11834144.html

themes/next/source/css/main.styl （对于有些版本的next对应的应该是custom.styl）

themes/next/source/css/_common/scaffolding/base.styl 定义了一些字体等等的样式

themes/next/source/css/_variables/base.styl 大局上定义了一些字体等等的样式，并且设成了样式全局变量。

更明显区分h1 h2 h3

打开 themes/next/source/css/main.styl （对于有些版本的next对应的应该是custom.styl）

行内代码样式更秀气

打开 themes/next/source/css/_common/scaffolding/base.styl

段距

打开 themes/next/source/css/_common/scaffolding/base.styl

行距

打开themes/next/source/css/_variables/base.styl

字体

打开themes/next/source/css/_variables/base.styl

打开主题的 _config.yml ，改为enable: true ，并且字体设置成 Noto Serif SC
